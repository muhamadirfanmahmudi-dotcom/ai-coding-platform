# Plan: 平台内置代码版本控制系统

## 目标
为 AI 代写平台添加 Git 风格的代码版本管理功能，开发者无需外部 Git 账号即可在平台内保存、追溯、对比代码历史。

## 核心设计

### 数据模型（3 张新表）

**projects** — 代码仓库/项目
```
id TEXT PRIMARY KEY,
name TEXT NOT NULL,
description TEXT,
owner_id TEXT NOT NULL (FK → users),
order_id TEXT (FK → orders, 可选关联订单),
created_at, updated_at
```

**commits** — 提交快照（链式 parent）
```
id TEXT PRIMARY KEY,
project_id TEXT NOT NULL (FK → projects),
parent_id TEXT (FK → commits, null = 首次提交),
message TEXT NOT NULL,
author_id TEXT NOT NULL (FK → users),
created_at
```

**files** — 每个 commit 的完整文件快照
```
id TEXT PRIMARY KEY,
commit_id TEXT NOT NULL (FK → commits),
path TEXT NOT NULL (如 src/index.ts),
content TEXT NOT NULL,
UNIQUE(commit_id, path)
```

设计要点：
- 每个 commit 存储**全量文件快照**（非 diff），简化读取历史版本
- parent_id 链形成提交历史链
- project_id + path 可查询某文件的完整变更历史

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/projects` | 创建项目 |
| GET | `/api/projects` | 列出我的项目 |
| GET | `/api/projects/:id` | 项目详情（含最新 commit） |
| POST | `/api/projects/:id/commit` | 提交代码快照（{ message, files: [{path, content}] }） |
| GET | `/api/projects/:id/commits` | 提交历史 |
| GET | `/api/projects/:id/commits/:commitId` | 查看某次提交详情 + 文件列表 |
| GET | `/api/projects/:id/tree/:commitId` | 获取某次提交的完整文件树 |
| GET | `/api/projects/:id/blob/:commitId/*` | 获取某次提交中某个文件内容 |
| GET | `/api/projects/:id/diff/:commitId1/:commitId2` | 两次提交的差异对比 |

### 实现范围（MVP）

1. **后端**：新路由 `src/routes/projects.ts` + 数据库迁移
2. **前端**：项目管理页面 + 提交历史查看 + 文件内容查看
3. **不包含**：分支、合并、冲突解决、tag（后续迭代）

### 文件变更清单

- `src/services/database.ts` — 新增 3 张表的 migrate + CRUD 方法
- `src/routes/projects.ts` — 新建，所有版本控制 API
- `src/index.ts` — 注册新路由
- `src/models/index.ts` — 新增类型定义
- `frontend/index.html` — 新增项目管理 UI

### 验证方案

1. 启动服务器 `npm run dev`
2. 注册/登录 → 创建项目 → 提交代码 → 查看历史 → 查看文件内容 → 对比两次提交
3. 验证：同一项目多次提交后，可回溯任意版本的任意文件内容
