# Git HTTP Backend 实现计划

## 目标
为开发者提供标准 Git 仓库，无需挂梯子即可使用 git clone/push/pull 命令。

## 技术方案：Git Smart HTTP Protocol

### 架构图
```
开发者电脑                          你的平台服务器 (Port 3000)
    │                                      │
    │  git clone                           │
    │  http://server/git/{orderId}.git     │
    │─────────────────────────────────────►│
    │                                      ├── 检查认证 (SID Token)
    │                                      ├── 读取 bare repo
    │  ◄───────────────────────────────────┤
    │                                      │
    │  git push origin main               │
    │─────────────────────────────────────►│
    │                                      ├── 检查认证
    │                                      ├── 更新 bare repo
    │                                      ├── 同步到 SQLite 数据库
    │  ◄───────────────────────────────────┤
```

## 实现步骤

### 步骤 1：创建 Git 仓库管理模块

**文件**: `src/services/git-repo.ts`

功能：
- `createBareRepo(orderId)` - 创建裸 Git 仓库
- `getBareRepo(orderId)` - 获取裸仓库路径
- `repoExists(orderId)` - 检查仓库是否存在
- `syncToDatabase(orderId)` - 将 Git 提交同步到 SQLite

存储路径：`repos/{orderId}.git/`

### 步骤 2：实现 Git Smart HTTP 协议

**文件**: `src/routes/git.ts`

端点：
1. `GET /git/:orderId.git/info/refs` - 仓库发现
2. `POST /git/:orderId.git/git-upload-pack` - 克隆/拉取
3. `POST /git/:orderId.git/git-receive-pack` - 推送
4. `GET /git/:orderId.git/HEAD` - 获取 HEAD 引用

### 步骤 3：认证中间件

**文件**: `src/middleware/git-auth.ts`

- 从 URL 参数或 HTTP Header 提取 SID
- 验证用户身份和订单权限
- 只有开发者可以 push，只有买家和开发者可以 clone

### 步骤 4：Git 命令执行器

**文件**: `src/services/git-commands.ts`

封装 Git 命令：
- `git init --bare` - 初始化裸仓库
- `git upload-pack` - 处理克隆/拉取
- `git receive-pack` - 处理推送
- `git update-server-info` - 更新服务器信息

### 步骤 5：注册路由

**修改文件**: `src/index.ts`

添加 Git 路由到 Express 应用。

## 使用方式

### 开发者克隆仓库
```bash
git clone http://your-server:3000/git/{orderId}.git \
  -c http.extraHeader="Authorization: Bearer <sid>"
```

### 开发者推送代码
```bash
cd {orderId}
git add .
git commit -m "feat: 添加登录功能"
git push origin main \
  -c http.extraHeader="Authorization: Bearer <sid}"
```

### 浏览器访问（只读）
```
http://your-server:3000/git/{orderId}.git/info/refs?service=git-upload-pack
```

## 验证方案

1. **单元测试**：测试仓库创建、认证、权限
2. **集成测试**：使用 git 命令测试完整流程
3. **手动测试**：
   - 创建开发者账户
   - 接单并初始化仓库
   - 使用 git clone 克隆
   - 修改文件并 push
   - 验证数据库同步

## 注意事项

1. **Windows 兼容性**：使用 Node.js child_process 执行 Git 命令
2. **并发控制**：同一仓库同时 push 时需要锁定
3. **错误处理**：Git 命令失败时返回标准 Git HTTP 错误格式
4. **日志记录**：记录所有 Git 操作用于审计
